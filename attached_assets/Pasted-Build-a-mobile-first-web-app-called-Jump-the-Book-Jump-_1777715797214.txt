Build a mobile-first web app called Jump the Book.

Jump the Book is an immersive visual reading companion that lets users step inside the book they are reading or listening to. It does not replace Kindle, Audible or physical books. Instead, it acts as a companion layer that helps users experience a story visually through scenes, characters, locations, mood, comic-style panels, chapter summaries and spoiler-safe visual guidance.

The app should be designed for a Replit Build Marathon MVP. It must be polished, demo-ready, mobile-first and visually impressive, but it should not try to build every future feature at once.

Important product principle:
The MVP must not directly extract or process full Kindle or Audible content. Instead, it should support public domain demo books, manually added current reads, and user-owned writing or pasted excerpts. Add placeholder “Connect Kindle” and “Connect Audible” buttons as Coming Soon features.

The core concept:
Users can select a public domain demo book, add a book they are currently reading, or upload/paste their own writing. The app then creates a visual story companion with chapter summaries, scene cards, character tracking, location moodboards and immersion mode.

Backend API:
Create Express endpoints:

GET /api/books
Returns all books.

GET /api/books/:id
Returns one book with details.

GET /api/books/:id/chapters
Returns chapters for a book.

GET /api/chapters/:id/scenes
Returns scenes for a chapter.

GET /api/books/:id/characters
Returns characters for a book.

POST /api/library
Adds a manually entered current read.

POST /api/generate-scenes
Accepts:
- title
- author
- chapterTitle
- text
- visualStyle
- spoilerMode

Returns structured JSON:
{
  chapterSummary,
  scenes: [
    {
      title,
      summary,
      characters,
      location,
      mood,
      imagePrompt,
      narration
    }
  ],
  characters,
  locations
}

For now, mock the AI response. Add clear comments showing where OpenAI API integration can be added later using process.env.OPENAI_API_KEY.
